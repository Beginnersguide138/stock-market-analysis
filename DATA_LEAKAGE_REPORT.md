# データリーク調査レポート

調査日: 2026-03-05

## 概要

本プロジェクトの予測モデル群について、意図しないデータリーク（データ漏洩）の有無を調査しました。

## 1. 意図的なリーク（既知・README記載済み）

### カンニング特徴量 (`_cheat1d`)

- **場所**: `backtest_hybrid_rolling.py` L133-137
- **内容**: `shift(-1)` で翌日の値を特徴量として使用
- **目的**: ショック伝播効果の分析（研究目的）

```python
for f in base_features:
    df_all[f'{f}_cheat1d'] = df_all[f].shift(-1)
```

## 2. 意図しないリーク

### (A) USD/JPY Return の方向不一致 — 重大度: 高

`pct_change(-1)` は未来方向の変動率を計算しており、意図せず翌日の為替情報がリークしています。

| ファイル | コード | 方向 | 状態 |
|----------|--------|------|------|
| `predict_micro_future.py` | `pct_change(-1)` | 未来方向 | **リーク** |
| `predict_ml.py` | `pct_change(-1)` | 未来方向 | **リーク** |
| `predict_micro_hybrid.py` | `pct_change(1)` | 過去方向 | 正常 |

**修正方法**: `pct_change(-1)` → `pct_change(1)` に変更

### (B) 特徴量エンジニアリングが分割前に実行 — 重大度: 高

全モデルで、テクニカル指標（EMA, MACD, 一目均衡表, RSI等）がデータ全体に対して計算された後、train/testに分割されています。

特に一目均衡表の先行スパン（SpanA/SpanB）は26日先にシフトするため、テスト期間のデータが訓練期間の特徴量に漏れる可能性があります。

**修正方法**: 分割後に各セットで個別に特徴量を計算する

### (C) バリデーション未実装 — 重大度: 高

大半のモデルで適切なバリデーションが行われていません。`TimeSeriesSplit` を使用しているのは以下の2ファイルのみです:

- `predict_micro_advanced.py`
- `predict_micro_cv.py`

**修正方法**: 全モデルに `TimeSeriesSplit` を導入

### (D) `ffill()` がデータ全体に適用 — 重大度: 中

```python
df_sansan.ffill(inplace=True)  # 全データに対して前方補完
```

**修正方法**: train/test 分割後に個別適用

### (E) train/test間にギャップがない — 重大度: 中

訓練データの最終日とテスト対象日の間にギャップがなく、特徴量（当日Close等）とターゲット（翌日Close）間の相関が不自然に高くなる可能性があります。

**修正方法**: train/test 間に1日以上のギャップを設ける

## 3. 影響度サマリー

| リーク種別 | 該当ファイル数 | 重大度 | 意図的 |
|-----------|-------------|--------|--------|
| カンニング特徴量 (`_cheat1d`) | 1 | 致命的 | はい |
| `pct_change(-1)` 為替リーク | 2 | 高 | いいえ |
| 特徴量計算が分割前 | 全ファイル | 高 | いいえ |
| バリデーション未実装 | 大半 | 高 | いいえ |
| `ffill()` が分割前 | 全ファイル | 中 | いいえ |
| train/testギャップなし | 全ファイル | 中 | いいえ |

## 4. 推奨アクション（優先度順）

1. `pct_change(-1)` → `pct_change(1)` に統一
2. 特徴量計算を train/test 分割後に実施
3. 全モデルに `TimeSeriesSplit` を導入
4. train/test 間に1日以上のギャップを設定
5. `ffill()` を分割後に個別適用
